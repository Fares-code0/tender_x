# مصفوفة الأدوار والصلاحيات

## مقدّمة

تحدّد هذه الوثيقة الأدوار الخمسة في نظام المناقصات والصلاحيات المرتبطة بكل دور. كل دور له مجموعة محددة من الإجراءات التي يمكنه القيام بها.

**المصادر:**
- `apps/api/src/middleware/auth.ts` — وسيط التحقق من الأدوار
- `apps/api/src/routes/*` — نقاط نهاية API وتنفيذ الصلاحيات
- `apps/web/src/lib/labels.ts` — تسميات الأدوار

---

## الأدوار الخمسة

| الدور | الكود | الوصف |
|------|------|-------|
| مسؤول النظام | ADMIN | يدير المستخدمين والأدوار والإعدادات العامة |
| مراجع الجودة | QA | يراجع المناقصات الجديدة ويقرر التحويل أو الرفض |
| كاتب العروض | WRITER | يُحضّر العروض والمرفقات ويرسلها للاعتماد |
| المدير | MANAGER | يعتمد العروض وينسّق التقديم وينسجل النتائج |
| المالك | OWNER | يراقب فقط، قراءة بلا تعديل |

## انتقالات الحالات حسب الدور

### مراجع الجودة (QA)
```
NEW ──→ UNDER_REVIEW (تلقائيًا عند التسجيل)
UNDER_REVIEW ──→ PROPOSAL_PREPARATION (عند اكتمال Checklist)
UNDER_REVIEW ──→ REJECTED (عند الرفض مع سبب)
```

### كاتب العروض (WRITER)
```
PROPOSAL_PREPARATION ──→ PENDING_APPROVAL (إرسال للاعتماد)
```

### المدير (MANAGER)
```
PENDING_APPROVAL ──→ SUBMITTED (اعتماد والتقديم)
PENDING_APPROVAL ──→ PROPOSAL_PREPARATION (إعادة بملاحظات)
PENDING_APPROVAL ──→ REJECTED (رفض)
SUBMITTED ──→ WON (تسجيل فوز)
SUBMITTED ──→ LOST (تسجيل خسارة)
```

### المالك (OWNER)
```
❌ لا يمكنه تغيير الحالات (قراءة فقط)
```

### مسؤول النظام (ADMIN)
```
❌ لا يشارك في سير عمل المناقصات
```
