# جرد الشاشات ومواصفاتها

## مقدّمة

توثّق هذه الوثيقة جميع الشاشات (الواجهات) في نظام المناقصات. كل شاشة لها:
- معرّف فريد (SCR-xx)
- ملف التطبيق
- من يراها (الأدوار المسموح لها)
- الحقول والعناصر
- الرحلات المرتبطة
- حالات العرض (فارغ، تحميل، نجاح، خطأ، عدم صلاحية، انتهاء جلسة)

## مواصفات الشاشات المفصّلة

### SCR-01: تسجيل الدخول

**الملف:** `LoginPage.tsx`  
**من يراها:** الكل (قبل التصريح)  
**الدور:** —

#### الحقول
- **البريد الإلكتروني** (إلزامي): حقل نصي
- **كلمة المرور** (إلزامي): حقل كلمة مرور
- **تذكرني** (اختياري): مربع اختيار

#### الرحلات المرتبطة
- JRN-07: متابعة / قراءة (بعد الدخول)

#### حالات العرض

| الحالة | السلوك |
|--------|-------|
| **فارغ** | نموذج فارغ، زر "دخول" معطّل |
| **تحميل** | عرض مؤشر تحميل، الحقول معطّلة |
| **نجاح** | إعادة توجيه إلى الرئيسية (SCR-02) |
| **خطأ** | رسالة: "بيانات الدخول غير صحيحة" |
| **انتهاء الجلسة** | رسالة: "جلسة غير صالحة، سجّل الدخول مجددًا" |

### SCR-02: الرئيسية / لوحة المعلومات

**الملف:** `HomePage.tsx`  
**من يراها:** الكل (حسب الدور)  
**الأدوار:**
- QA: مناقصات قيد المراجعة
- WRITER: مناقصات في انتظار الإعداد
- MANAGER: مناقصات في انتظار الاعتماد والتقديم
- OWNER: ملخص إحصائي (قراءة فقط)
- ADMIN: ملخص النظام

#### الحقول / العناصر
- **رقم المناقصة** (معرّف)
- **العنوان**
- **الحالة الحالية** (NEW, UNDER_REVIEW, إلخ)
- **الجهة المعلنة**
- **موعد الإغلاق**
- **المسؤول الحالي**
- **رسم بياني**: توزيع المناقصات حسب الحالة (للمالك)

#### الرحلات المرتبطة
- JRN-01: تسجيل مناقصة (انتقال إلى SCR-05)
- JRN-02, JRN-03, JRN-04: من القائمة إلى التفاصيل
- JRN-07: متابعة وقراءة

#### حالات العرض

| الحالة | السلوك |
|--------|-------|
| **فارغ** | رسالة: "لا توجد مناقصات" |
| **تحميل** | مؤشر تحميل + مخطط هيكلي |
| **نجاح** | عرض البطاقات / الجدول |
| **خطأ** | رسالة: "حدث خطأ في تحميل البيانات" |
| **عدم صلاحية** | رسالة: "لا توجد صلاحية" |
| **انتهاء الجلسة** | إعادة توجيه إلى SCR-01 |

### SCR-03: قائمة المناقصات

**الملف:** `TendersPage.tsx`  
**من يراها:** الكل  
**الأدوار:** القراءة: الكل | التعديل: QA/MANAGER/ADMIN

#### الحقول / العناصر
- **رقم المناقصة**
- **العنوان**
- **الحالة**
- **الجهة**
- **موعد الإغلاق**
- **المسؤول**

#### المرشحات (Filters)
- **الحالة** (dropdown): NEW, UNDER_REVIEW, PROPOSAL_PREPARATION, PENDING_APPROVAL, SUBMITTED, REJECTED, WON, LOST
- **الجهة** (بحث نصي)
- **المسؤول** (dropdown)
- **موعد الإغلاق** (نطاق تاريخ)
- **البحث العام** (q): عنوان أو وصف

#### الترتيب والترقيم
- **الفرز الافتراضي:** موعد الإغلاق (الأقرب أولاً)
- **الترقيم:** 20 صف في الصفحة
- **الملاحة:** السابق / التالي / الذهاب إلى الصفحة

#### الرحلات المرتبطة
- JRN-02: مراجعة QA (انقر على صف → SCR-04)
- JRN-03: إعداد العرض (انقر على صف → SCR-04)
- JRN-04: اعتماد المدير (انقر على صف → SCR-04)
- JRN-07: متابعة المالك (قراءة فقط)

#### حالات العرض

| الحالة | السلوك |
|--------|-------|
| **فارغ** | رسالة: "لا توجد مناقصات مطابقة" |
| **تحميل** | مؤشر تحميل على الجدول |
| **نجاح** | عرض الجدول مع الصفوف |
| **خطأ** | رسالة: "فشل تحميل البيانات" |
| **عدم صلاحية** | رسالة: "ليست لديك صلاحية لهذا الإجراء" |
| **انتهاء الجلسة** | إعادة توجيه إلى SCR-01 |

### SCR-04: تفاصيل المناقصة

**الملف:** `TenderDetailsPage.tsx`  
**من يراها:** الكل  
**الأدوار:** القراءة: الكل | التعديل: QA/MANAGER/ADMIN (حسب الحالة)

#### الحقول
- **رقم المناقصة** (معرّف فريد)
- **العنوان**
- **الوصف**
- **الجهة المعلنة** (entity)
- **موعد الإغلاق** (closingDate)
- **المصدر** (source URL)
- **الحالة الحالية** (status)
- **المسؤول** (currentAssigneeId)
- **اعتماد المدير** (managerApprovedAt) — قراءة فقط

#### التقسيمات (Tabs)
1. **التفاصيل**: المعلومات الأساسية
2. **Checklist**: عناصر الـ Checklist (QA يملأها)
3. **المرفقات**: الملفات المرفقة (WRITER يرفعها)
4. **السجل**: سجل الحالات وتاريخ التعديلات (Audit Log)

#### الرحلات المرتبطة
- JRN-02: مراجعة QA + Checklist
- JRN-03: إعداد العرض + المرفقات
- JRN-04: اعتماد المدير
- JRN-05: تقديم رسمي
- JRN-06: تسجيل النتيجة
- JRN-07: قراءة (OWNER)

#### حالات العرض

| الحالة | السلوك |
|--------|-------|
| **تحميل** | مؤشر تحميل + مخطط هيكلي |
| **نجاح** | عرض التفاصيل الكاملة |
| **خطأ** | رسالة: "فشل تحميل المناقصة" |
| **عدم وجود** | رسالة: "المناقصة غير موجودة" |
| **عدم صلاحية** | رسالة: "ليست لديك صلاحية لهذا الإجراء" |
| **انتهاء الجلسة** | إعادة توجيه إلى SCR-01 |

### SCR-05: نموذج المناقصة

**الملف:** `TenderFormPage.tsx`  
**من يراها:** مراجع الجودة (QA)  
**الأدوار:** QA فقط

#### الحقول (مع التحقق من الصحة)
- **العنوان** (إلزامي): نص، BR-010 ✅
- **الجهة المعلنة** (إلزامي): dropdown أو بحث، BR-010 ✅
- **موعد الإغلاق** (إلزامي): منتقِي تاريخ + وقت، BR-010 ✅
- **المصدر** (اختياري): URL
- **الوصف** (اختياري): نص طويل
- **فحص التكرار**: تحذير تلقائي + خيار `force=1`

#### الأزرار
- **حفظ**: إنشاء المناقصة (POST /tenders)
- **إلغاء**: العودة للقائمة
- **مسح**: تفريغ النموذج

#### الرحلات المرتبطة
- JRN-01: تسجيل مناقصة جديدة

#### حالات العرض

| الحالة | السلوك |
|--------|-------|
| **فارغ** | نموذج فارغ، الزر معطّل حتى ملء الإلزامي |
| **تحميل** | عند الحفظ: مؤشر تحميل |
| **نجاح** | رسالة: "تم إنشاء المناقصة بنجاح" → إعادة توجيه إلى SCR-04 |
| **خطأ** | رسالة: "فشل الإنشاء: [السبب]" (مثلاً: بيانات ناقصة) |
| **تحذير** | إذا وجد تكرار: "هذه المناقصة موجودة. هل تريد المتابعة؟" |
| **عدم صلاحية** | رسالة: "ليست لديك صلاحية" → إعادة توجيه |
