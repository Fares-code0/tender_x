# حزمة توثيق UX/Design — نظام المناقصات

<div dir="rtl">

توثّق هذه الحزمة قواعد العمل، الأدوار والصلاحيات، رحلات المستخدمين، وجرد الشاشات لنظام إدارة المناقصات (Smart Tender Management System)، ليكون مرجعًا موحّدًا للفريقين التقني والتصميمي.

> **آخر مزامنة مع الكود: نهاية M8 (سجل العمليات + التقوية) — الإصدار v1.0 — 2026-07-22.** الحزمة تعكس المشروع مكتملًا (M0…M8). كل القواعد BR-001/002/004/005/008/009/010/011 والإجراءات ACT-01…ACT-16 "منفّذة ✅". يشمل ذلك: المراجعة والـChecklist (M3)، سير العمل الكامل عبر State Machine مركزية (M4)، **المرفقات** خلف `StorageService` (M5، ACT-13)، **الإشعارات** (`NotificationService` + job الإغلاق BR-009 + جرس الهيدر — M6، ACT-14)، **لوحات المعلومات والتقارير** (Recharts + تصدير CSV — M7، ACT-15، SCR-02/SCR-08)، و**سجل العمليات** للقراءة فقط + التقوية الأمنية (Helmet/CORS/rate-limit) (M8، ACT-16، NFR-005). لم تبقَ إجراءات دورة حياة مؤجَّلة.

> **تحديثات ما بعد v1.0:** أُغلقت الفجوات الواجهية الموثّقة سابقًا: (1) **إدارة كاملة للمستخدمين** من SCR-06 (إنشاء/تعطيل/تغيير دور)، (2) **فلاتر قائمة المناقصات** (المسؤول/نطاق تاريخ الإغلاق/الترتيب) في SCR-03، (3) شاشة **إعدادات النظام** SCR-09 لتعديل أيام تنبيه الإغلاق (ACT-17، BR-009)، (4) **قفل تعديل** المناقصة بعد التقديم/الإغلاق (ACT-02، `TENDER_LOCKED`). الإجراءات الآن ACT-01…ACT-17.

## اصطلاح الترميز

- ✅ **مكتمل / منفّذ**: المستند مكتوب بالكامل، أو القاعدة/الميزة مطبَّقة فعليًا في الكود.
- 🔷 **مخطّط**: المستند لم يُكتب بعد، أو القاعدة/الميزة موثّقة كمتطلب دون تنفيذ حالي.

## المعرّفات المشتركة

المعرّفات التالية مشتركة بين مستندات الحزمة، ويجب استخدامها بشكل متّسق:

| البادئة | المعنى |
|---|---|
| `BR-xxx` | قاعدة عمل (Business Rule) — معرَّفة في المستند 01 |
| `ACT-xxx` | إجراء/فعل يقوم به مستخدم في النظام — يُستخدم في مصفوفة الأدوار (02) وجدول الانتقالات (01) |
| `SCR-xxx` | شاشة في الواجهة — تُعرَّف في جرد الشاشات (04) |
| `JRN-xxx` | رحلة مستخدم كاملة — تُعرَّف في مستند رحلات المستخدمين (03) |
| `UAT-xxx` | حالة اختبار قبول المستخدم — تُعرَّف في خطة الـUAT (05)، وتُتبَّع إلى BR/ACT/SCR/JRN والكود |
| `FR-xxx` | متطلب وظيفي — يُعرَّف في الـSRS (06)، مجمّع حسب الوحدة ومربوط بـendpoint/BR/ACT |
| `NFR-xxx` | متطلب غير وظيفي — يُعرَّف في الـSRS (06)؛ ومنه `NFR-005` (سجل العمليات للقراءة فقط) المُشار إليه في 01/02/03 |
