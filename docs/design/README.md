# حزمة توثيق UX/Design — نظام المناقصات

<div dir="rtl">

توثّق هذه الحزمة قواعد العمل، الأدوار والصلاحيات، رحلات المستخدمين، وجرد الشاشات لنظام إدارة المناقصات (Smart Tender Management System)، ليكون مرجعًا موحّدًا للفريقين التقني والتصميمي.

> **آخر مزامنة مع الكود: نهاية M6 (الإشعارات) — 2026-07-22.** حُدِّثت الحزمة لتعكس تنفيذ M3 (المراجعة والـChecklist)، M4 (سير العمل الكامل)، **M5 (المرفقات)**، و**M6 (الإشعارات)**. انتقلت القواعد BR-001/002/004/005/009/011 والإجراءات ACT-04…ACT-14 إلى "منفّذ ✅". أُضيف **نظام المرفقات** (رفع/قائمة/تحميل خلف `StorageService`، قيود النوع والحجم، versioning، وتبويب "المرفقات" في SCR-04 — ACT-13)، و**نظام الإشعارات** (`NotificationService` مربوط بستة أحداث سير عمل، وjob تنبيه اقتراب الإغلاق عبر node-cron وفق BR-009، ومسارات `/notifications`، وجرس إشعارات في الهيدر — ACT-14). لم تبقَ إجراءات دورة حياة مؤجَّلة.

## اصطلاح الترميز

- ✅ **مكتمل / منفّذ**: المستند مكتوب بالكامل، أو القاعدة/الميزة مطبَّقة فعليًا في الكود.
- 🔷 **مخطّط**: المستند لم يُكتب بعد، أو القاعدة/الميزة موثّقة كمتطلب دون تنفيذ حالي.

## المعرّفات المشتركة

المعرّفات التالية مشتركة بين مستندات الحزمة الأربعة، ويجب استخدامها بشكل متّسق:

| البادئة | المعنى |
|---|---|
| `BR-xxx` | قاعدة عمل (Business Rule) — معرَّفة في المستند 01 |
| `ACT-xxx` | إجراء/فعل يقوم به مستخدم في النظام — يُستخدم في مصفوفة الأدوار (02) وجدول الانتقالات (01) |
| `SCR-xxx` | شاشة في الواجهة — تُعرَّف في جرد الشاشات (04) |
| `JRN-xxx` | رحلة مستخدم كاملة — تُعرَّف في مستند رحلات المستخدمين (03) |
